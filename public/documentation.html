About

This project is a RESTful API serving a request back to the user via the server and business logic. The main purpose of the API is to support a database for animated films, once connected to a client and server. This API contains the necessary logic to ensure the data is transferred. Right now the data is hardcoded for testing purposes.

Endpoints

Request URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of ALL movies to the user /movies GET none
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user; /movies/:title GET none
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/:genre GET none
Return data about a director (bio, birth year, death year) by name /movies/:director/:name GET none
Allow new users to register /users PUT JSON formatted user info: { 'user':'username', 'photo': 'photurl' 'birthdate': 'YYYY-MM-DD', 'favorites': {} }
Allow users to update their user info (username) /users/user PATCH
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/favorites/:title PATCH
Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed —- more on this later) /users/favorites/:title DELETE
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later). /users/:id DELETE
